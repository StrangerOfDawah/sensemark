> Sensemark переводит **на русский с автоматически определяемого исходного языка** — выбирать язык вручную не нужно.

Выделение на русском игнорируется без открытия карточки и запроса к API — в том числе когда внутри есть небольшие технические вставки вроде `dev-секрет`, `APP_ENV`, URL или название ChatGPT. Если в одном выделении действительно смешаны русское и иностранное предложения, русские фрагменты остаются без изменений, а остальные переводятся и показываются отдельными секциями с названием исходного языка.

Перед запросом Sensemark локально удаляет скрытые дубликаты, кнопки, подписи интерфейса и управляющие Unicode-символы. На Quran.com специальные шрифтовые глифы заменяются уже встроенной в страницу простой арабской копией. Поэтому модель получает настоящий текст, а не визуальные коды, и лишний DOM-шум не расходует токены.

<br>

## Перевод не заставляет ждать

Ответ идёт потоком по SSE: карточка открывается вместе с первыми буквами перевода, пока модель ещё дописывает остальное. Пустого окна загрузки перед началом ответа нет.

<img src="docs/feature-stream.png" alt="Три стадии перевода: текст появляется по мере генерации">

Закрыли карточку посреди перевода — запрос обрывается, недогенерированные токены не оплачиваются.

<br>

## Одно слово — нужное значение

Выделили одно слово? Расширение само подхватит предложение вокруг и попросит перевод именно в этом смысле. Под основным переводом покажет остальные частые значения — на случай, если нужно было другое.

Если выделенный фрагмент не является общеупотребительным словом — например, это название, имя, никнейм или опечатка — Sensemark не станет придумывать перевод. Карточка переключится в отдельный янтарный режим **«Объяснение»**, покажет категорию фрагмента и кратко опишет по написанию и контексту, что это может быть. Если определить не получится, аккуратно укажет, что это, вероятно, имя или название.

<img src="docs/feature-context.png" alt="Слово bank в двух контекстах: банк и берег">

Карточка живёт в Shadow DOM, поэтому стили сайта её не ломают, а тема подхватывается автоматически.

## Установка

1. Скачайте архив со страницы [**Releases**](https://github.com/StrangerOfDawah/sensemark/releases/latest) и распакуйте
2. Откройте `chrome://extensions`
3. Включите **Режим разработчика** — тумблер справа сверху
4. Нажмите **Загрузить распакованное расширение** и выберите распакованную папку

Папку после этого нельзя удалять или переименовывать — Chrome загружает расширение прямо из неё. Положите её туда, где она останется надолго.

До публикации в Chrome Web Store расширение устанавливается вручную. Chrome будет показывать напоминание про режим разработчика при запуске — это нормально для расширений, установленных таким способом.

<br>

## Тестовая сборка из PR

Для каждого pull request GitHub Actions проверяет код и прикрепляет распаковываемую тестовую сборку. Откройте нужный запуск **PR test build** на вкладке Actions, скачайте артефакт `sensemark-pr-N`, распакуйте его и загрузите полученную папку через `chrome://extensions` → **Загрузить распакованное расширение**.

Артефакт хранится 14 дней и не содержит API-ключей или пользовательских настроек. Для проверки удобно использовать отдельный профиль Chrome, чтобы тестовая версия не пересекалась с установленным релизом. Workflow также можно запустить вручную через **Run workflow**.

<br>

## Ключ OpenAI

Нужен ключ OpenAI API. Это **не** подписка ChatGPT Plus — она программного доступа не даёт, ключ оплачивается отдельно по факту использования.

1. Заведите ключ на [platform.openai.com/api-keys](https://platform.openai.com/api-keys) и пополните баланс
2. Нажмите на иконку расширения — откроются настройки
3. Вставьте ключ и нажмите **Проверить ключ**

**Про деньги.** По умолчанию используется экономичная модель `gpt-4o-mini`. Стоимость зависит от действующих тарифов OpenAI. Расход видно на [platform.openai.com/usage](https://platform.openai.com/usage), там же можно настроить лимиты.

## Как пользоваться

| Способ | Как |
| --- | --- |
| Горячая клавиша | Выделить текст → <kbd>⌘</kbd><kbd>⇧</kbd><kbd>Y</kbd> (Mac) или <kbd>Ctrl</kbd><kbd>⇧</kbd><kbd>Y</kbd> (Windows) |
| Контекстное меню | Выделить текст → правый клик → «Перевести на русский» |
| Автоматически | Включить тумблер в настройках — переводит по любому выделению мышью |
| Масштаб | <kbd>⌘</kbd>/<kbd>Ctrl</kbd> + колесо над карточкой |

В карточке: кнопка-иконка копирует перевод, «Оригинал» разворачивает исходный текст. Закрыть — <kbd>Esc</kbd>, крестик, клик мимо или скролл.

Горячую клавишу можно переназначить на `chrome://extensions/shortcuts`. Если она не сработала сразу — проверьте там, что комбинация назначена: Chrome молча оставляет поле пустым, когда сочетание уже занято другим расширением.

<br>

## Устройство

| Файл | Назначение |
| --- | --- |
| `manifest.json` | Манифест, разрешения, горячая клавиша |
| `background.js` | Service worker: контекстное меню, стриминг из OpenAI, кэш переводов |
| `language-detection.js` | Локальная проверка: нужно ли переводить выделенный текст |
| `selection-text.js` | Очистка выделения и восстановление семантического текста из шрифтовых глифов |
| `word-response.js` | Разбор ответа для слова: обычный перевод или объяснение имени/названия |
| `text-response.js` | Разбор обычного и многоязычного перевода по секциям |
| `content.js` | Карточка на странице, разбор контекста, масштаб и размер |
| `options.html` · `options.js` | Страница настроек |
| `icons/` | Иконки 16–128 |

Ключ хранится в `chrome.storage.local`. Выделенный текст и, для короткого фрагмента, окружающее предложение отправляются напрямую в OpenAI для перевода. У расширения нет сервера разработчика и аналитики; подробности — в [политике конфиденциальности](PRIVACY.md).

Повторные переводы одного фрагмента берутся из кэша в памяти service worker (200 последних) и не тратят токены. Ограничение на длину — 5000 символов за раз, чтобы случайный <kbd>⌘</kbd><kbd>A</kbd> не ушёл в API целой страницей.

Если выделенный фрагмент существует только как изображение и у страницы нет иностранного текстового слоя, Sensemark показывает локальное сообщение и ничего не отправляет в API. Автоматический OCR намеренно не запускается: он увеличил бы стоимость и мог бы исказить декоративный или коранический текст.

<br>

## Ограничения

- Работает только там, где Chrome разрешает расширениям выполнять скрипты: на `chrome://`, в Chrome Web Store и на страницах других расширений карточка не появится
- Текст внутри PNG, фотографии или canvas нельзя перевести обычным выделением; расширение не отправляет такие изображения в OpenAI и предлагает выбрать доступную текстовую подпись
- Язык перевода зафиксирован русским — значение подставляется в системный промпт, поэтому свободного ввода нет намеренно. Меняется в `DEFAULTS` в `background.js` и `options.js`
- После правок кода нужно нажать «Обновить» на карточке расширения в `chrome://extensions` и перезагрузить открытые вкладки
